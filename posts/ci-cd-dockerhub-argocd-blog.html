CI/CD with Docker Hub and ArgoCD

Overview

This guide explains how to implement a GitOps-based CI/CD pipeline using GitHub Actions, Docker Hub, and ArgoCD for Kubernetes deployments.

  • Automatic service change detection
  • Parallel matrix builds
  • Docker image publishing
  • GitOps manifest updates
  • ArgoCD deployment synchronization

1. Create Docker Hub Repository

Create a private Docker Hub repository to store backend images.

Recommended format: your-username/project-name

2. Create Docker Hub Access Token

Docker Hub → Account Settings → Personal Access Token → Generate New Token

Copy and securely store the generated token.

3. Configure GitHub Secrets

Settings → Secrets and variables → Actions

Create the following repository secrets:

DOCKER_USERNAME = your docker hub username
DOCKER_PASSWORD = docker hub personal access token

4. CI/CD Flow

ci.yml
   ↓
detect changed services
   ↓
matrix build
   ↓
reusable-build.yml
   ↓
docker build/push
   ↓
update gitops
   ↓
ArgoCD sync deployment

5. Backend CI Workflow

The workflow automatically detects which backend services changed and rebuilds only those services. 


name: backend-ci

on:
  push:
    branches:
      - main
    paths:
      - backend/**
      - .github/workflows/backend-ci.yml
      - .github/workflows/_build-and-push-update.yml
  workflow_dispatch:

permissions:
  contents: write

jobs:
  detect-changes:
    runs-on: ubuntu-latest

    steps:
      - name: Check out repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0
Matrix builds allow multiple backend services to build in parallel.

6. Reusable Build Workflow

Reusable workflows help standardize Docker builds and reduce duplication. 


name: build-and-push-update

on:
  workflow_call:
    inputs:
      service_name:
        required: true
        type: string

jobs:
  build-push-update:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to Docker Hub
        uses: docker/login-action@v3

      - name: Build and Push Image
        uses: docker/build-push-action@v6

7. GitOps Update Strategy

After image builds complete successfully, the workflow updates: 

gitops/environments/dev/<service>/values.yaml

ArgoCD continuously monitors the GitOps repository and automatically synchronizes Kubernetes deployments.

Benefits:
  • Declarative deployments
  • Full deployment history
  • Easy rollback support
  • Environment consistency

8. Workflow Behavior

  • Only changed services are rebuilt
  • Multiple services build in parallel
  • GitOps manifests update automatically
  • ArgoCD deploys the latest image tags
  • workflow_dispatch builds all services

10. Conclusion

Combining GitHub Actions, Docker Hub, and ArgoCD creates a scalable GitOps-based deployment pipeline for microservices.

Matrix builds reduce CI execution time, while GitOps keeps deployments auditable and reproducible.
