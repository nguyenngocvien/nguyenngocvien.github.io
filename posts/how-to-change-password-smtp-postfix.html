How to Change SMTP Password in Postfix

Overview

This guide shows how to update the SMTP password stored in Postfix and reload the service so outgoing mail keeps working.

  • SSH into the mail server
  • Edit /etc/postfix/sasl_passwd
  • Regenerate the Postfix hash map
  • Set secure file permissions
  • Restart Postfix and verify SMTP delivery

1. Access the Server

First, log in to the server that has Postfix installed.

ssh username@server-ip
Replace username with your account name and server-ip with the server address.

2. Edit sasl_passwd

2.1 Open the file

Use your preferred editor to open the SMTP credential file.

sudo vi /etc/postfix/sasl_passwd

2.2 Update the password

Find the SMTP host entry and replace the password with the new value.

[smtp.yourmailserver.com]:587 your-username:new-password
  • Replace new-password with the updated password.
  • If the password contains special characters, wrap both the username and password in quotes.
[smtp.yourmailserver.com]:587 "your-username":"new-!@#$%^&*()_+password"

2.3 Save and exit

Press ESC, type :wq, then press Enter to save and quit.

3. Rebuild the Postfix Hash

After editing the file, regenerate the hash database.

sudo postmap /etc/postfix/sasl_passwd

4. Check File Permissions

Lock down the credential file so only root can read it.

sudo chmod 600 /etc/postfix/sasl_passwd /etc/postfix/sasl_passwd.db

5. Restart Postfix

Reload Postfix so the updated credentials take effect.

sudo systemctl restart postfix

6. Test the SMTP Connection

Confirm the SMTP server is reachable and mail delivery works.

telnet smtp.yourmailserver.com 587

Or send a test email:

echo "Test SMTP" | mail -s "SMTP Test" your-email@example.com

7. Check the Logs if Mail Fails

Review Postfix logs if authentication or delivery fails.

tail -f /var/log/mail.log
If you hit authentication errors, re-check the username, password, and Postfix configuration.

Summary

  1. Edit /etc/postfix/sasl_passwd.
  2. Run sudo postmap /etc/postfix/sasl_passwd.
  3. Set permissions with sudo chmod 600 /etc/postfix/sasl_passwd /etc/postfix/sasl_passwd.db.
  4. Restart Postfix with sudo systemctl restart postfix.
  5. Test mail delivery with telnet or mail -s.
